    • Software Engineering and Design Enhancement

    The original artifact for this enhancement came from CS-360 – Mobile Architecture and Programming. This artifact was chosen because it offered the opportunity to convert a mobile application created in Android Studio into a full-stack application using the MEAN stack. The enhancement in this narrative is the backend server application for the Inventory Management System. This component was designed to serve as the core API for the frontend to retrieve, update, and manage inventory items. It is an Express.js application built using Node.js, incorporating MongoDB for data storage and JWT for authentication. The server is intended to provide a series of RESTful API endpoints to interact with data. It was initially created for the first module of the course, but the current version reflects enhancements made throughout the course. The wireframe visualization represents the current state of the planned frontend component, as of this module. It shows the basic layout and components that will be used to interact with the server, and does not represent an application currently under development.

    The process of enhancing and modifying the server taught me several valuable lessons:

    • Importance of Planning: I learned that proper planning is key. Before adding a feature, it's important to properly design it, understand the inputs and outputs, and the complexity of the task. This makes the entire development process easier.

    • Value of Modularity: The improvements in modularity allowed the server to become better organized and maintainable, making it easier to expand with new features.

    • Balancing Functionality and Security: I saw that adding a new feature can create more security vulnerabilities if not properly designed with security in mind.

    • Debugging skills: It was key to understand how to use console logging and debugging to find bugs or problems.

    • Importance of Configuration: Understanding that configuration is key to managing complexity in applications by properly configuring secrets.

    • Iterative Process: Software development is truly an iterative process of build-test-improve. The current version is not the final version, and it can always be improved upon further.

    The challenges I faced primarily related to handling complex queries, properly implementing security, and organizing code in a modular way. However, these challenges helped me to grow as a developer.

    Enhancement 2

    Database Enhancement

    The artifact used for the Databases Enhancement was created in CS-360 – Mobile Architecture and Programming. The Inventory App in the class used SQLite since the application was created in Android Studio. This artifact was selected because in porting the original mobile application to a full-stack application, the database structure needs to be modified to MongoDB. The original work and the new version are similar in core functions, schema enforcement, connection management, and encapsulation. The full-stack version uses a Mongoose schema to define the structure of the Item and User documents, rather than a relational database structure like SQL. The MongoDB implementation also includes connection handling, retry logic, and graceful shutdown procedures which are more sophisticated than the SQLite implementation. Both implementations encapsulate database access within a single class/module, promoting separation of concerns.

    The database enhancements implemented through MongoDB and Mongoose created an environment that is more robust and flexible than the original. MongoDB is used industry-wide for full-stack applications in the MEAN stack. Declaring types and defaults in the documents provides a base layer of security for the database. When paired with the Algorithms enhancement, the Database enhancement mitigates potential vulnerabilities using authentication, authorization, and password hashing.
